Layout

The only HTML you need in the body of your doc one of these structure variations.

The structure you use may change depending on the state of your app or user.

Header and Tray are optional. Likely you will not use tray for an anonymous user, enabling it when logged in. Header content may change depending on login state. Tabs are generally used for switching within a set of related views.

Default

An empty view with no navigation

<div id="app">
  <div class="view">
  </div>
</div>

Header

Header contains your branding and top level navigation

<div id="app">
  <header>
  </header>
  <div class="view">
  </div>
</div>

Tray

Tray is a vertical sidebar on the left. It can be used as primary or secondary navigation

<div id="app">
  <tray>
  </tray>
  <div class="view has-tray">
  </div>
</div>

Tabs

Tabs provide for secondary navigation

<div id="app">
  <header>
  </header>
  <tabs>
  </tabs>
  <div class="view has-tabs">
  </div>
</div>

View

Most of your rendering will go in .view

.view is a absolute positioned div that fills the screen, minus room for the header, tabs and tray if required (that's what .has-tray does).

.content provides a fixed width container. .panel provised fixed width and white background.
